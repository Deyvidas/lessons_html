a { display: inline; <!-- Чтобы не занимать всю строчку. --> font-family: Montserrat; <!-- Шрифт. --> color: #0596cf; <!-- Цвет шрифта ссылки. --> font-size: 15px; <!-- Размер шрифта ссылки. --> text-decoration-color: #ff0000e5; <!-- Цвет подчеркивания ссылки. --> }


Помимо непременного href, тег a может иметь необязательный атрибут target
(англ. target, «мишень»). Он «нацеливает» браузер, указывая вкладку,
где нужно открыть страницу, на которую ведёт ссылка.
Без атрибута target она откроется в текущей вкладке.

Если добавлен атрибут target со значением “_blank” (от англ. blank, «чистый»),
то другая страница будет показана в новой, пустой вкладке. Так лучше поступать
со ссылками на внешние ресурсы. Тогда ваша страница останется открытой
в браузере пользователя, и вы не потеряете трафик.




Аттрибут id: <h1 id="unique_id">...</h1>



Аттрибут (id="unique_id") создает уникальную ссылку по которой можно обращаться из любого места на ТЕКУЩЕЙ странице.



код

<h1 id="start_page" class="h1_red">Оглавление</h1> <!-- Это первый элемент в теге <body> c id="start_page". --> ... какие то элементы <h1 id="start_page" class="h1_red">Оглавление2</h1> <!-- Это второй элемент в теге <body> id="start_page". --> <a href="#start_page" target="_parent"><img src="pictures/arrow_up1.png" alt="arrow_up1"></a>

реализация


Оглавление2




Теперь к этой картинке прикреплена ссылка на первый элемент с тэгом start_page,
и если создастся любой другой элемент, с таким же id то приоритетнее тот элемент
который обявлен раньше в текущем html коде.



Также, из одного файла html можно ссылаться на элемент находящийся в другом,
файле html. id должны быть уникальными только внутри одного файл html,
то есть в одном файле не должно существовать два одинаковых id,
а вот если id элемента в одном файле совпадает с id элемента другого файла
они друг другу не будут мешать.

Списки <ul><li>...</li></ul> | <ol><li></li></ol> | <dl><li>...</li></dl>

  • Маркированный список. Набор пунктов, очередность которых не­ важна.
  • Нумерованный список. Перечень, в котором последовательность элементов имеет значение.
  • Список определений. Состоит из пар «имя/значение», в том числе терминов и определений.

Все элементы списков, собственно списки и тексты пунктов являются блочными
по умолчанию, поэтому они начинаются с новой строки, а над элементом и под
ним остается пустое пространство, но это можно изменить с помощью CSS.


Маркированный список.


<!-- <ul type="disc | circle | square">...</ul> --> <ul> <li>Булгаков</li> <li>Достоевский</li> <li>Толстой</li> <li>Чехов</li> </ul>

Пример отображения в браузере:

  • Булгаков
  • Достоевский
  • Толстой
  • Чехов

Нумерованный список.


<!-- <ol type="A | a | I | i | 1">...</ol> --> <!-- <ol type="A | a | I | i | 1" reversed>...</ol> --> <!-- <ol type="1" start="number"></ol> аттрибут старт доступен только если type="1". --> <ol type="1" start="0" reversed> <li>Булгаков</li> <li>Достоевский</li> <li>Толстой</li> <li>Чехов</li> </ol>

Пример отображения в браузере:
